# file Buffer Plugin

The `file` buffer plugin provides persistent buffer implementation. It uses file to store buffer chunks.

### Example Config

    :::text
    <match pattern>
      buffer_type file
      buffer_path /var/log/fluent/myapp.*.buffer
    </match>

NOTE: See <a href="config-file">Config File</a> for the basic structure and syntax of the configuration file.

### Parameters

#### buffer_type (required)
The value must be `file`.

#### buffer_path (required)
Path to store buffer chunks. ‘*’ is replaced with random characters. This parameter is required.

#### buffer_chunk_limit
The size of each buffer chunk. Default is 8m. Suffixes “k” (KB), “m” (MB), “g” (GB) can be used. See [Buffer Plugin Overview](buffer-plugin-overview) for the basic buffer structure. 

#### buffer_queue_limit
The length of the chunk queue. See [Buffer Plugin Overview](buffer-plugin-overview) for the basic buffer structure. Default is 256.

#### flush_interval
The interval of the data flush. Suffixes “s” (seconds), “m” (minutes), “h” (hours) can be used. Sometimes this parameter is ignored by specifying `time_slice_format` parameter.

#### retry_wait
The interval between retries. Suffixes “s” (seconds), “m” (minutes), “h” (hours) can be used.
